Sidlayout

Beskrivning av begrepp

Webbsideslayout (fixerad, flytande och elastisk) (fixed, fluid, and elastic)

Tanken med att skapa en webbsideslayout med CSS är att den ska kunna fungera någorlunda bra oberoende av vilken webbläsare och hårdvara (smartphone, läsplatta, stationär dator) besökaren av webbplatsen använder. De flesta webbsidesdesigner kan kategoriseras som antingen fixerade eller flytande. En tredje vanligt förekommande variant är elastiska designers.

Fixerad layout - Denna layout ger störst kontroll över hur webbsidan kommer att se ut, men för vissa besökare kan det bli svårt att hantera sidan, tex de som använder en väldigt liten skärm. En webbsidas innehåll behåller samma storlek oberoende av webbfönstrets storlek. Ofta centreras innehållet, men det förekommer också att det placeras längs vänsterkanten. Här används pixel-baserade enheter för att definiera sidans dimensioner.

Flytande layout - Denna layout anpassas efter webbläsarens fönster, genom att växa eller krympa då fönstret blir större respektive mindre. En sådan design kräver större utmaningar vid skapandet av webbsidan, för att den ska se bra ut för alla möjliga skärmstorlekar. Här används procent-baserade enheter för att definiera sidans dimensioner.

Elastisk layout - Denna design är en kombination av fixerad oh flytande design. Designen anpassar sig efter den textstorlek som besökarens webbläsare har. Här används em-baserade enheter för att definiera sidans dimensioner.

Fördelar med att använda CSS istället för tabeller

Då webben var nyuppfunnen, var det vanligt att man använde sig av tabeller skapade med taggen <table> för att organisera webbsidors innehåll. Taggen fick dock ofta manipuleras på mer eller mindre konstiga sätt, eftersom att den inte är tänkt att användas för layout. Till exempel kunde man placera en tabell inuti en cell hos en annan tabell. Andra nackdelar med den här metoden är också att det adderas en mängd extra HTML-kod, och att den är svår att modifiera i efterhand. Med CSS-design använder man istället individuella taggar för det innehåll som man vill placera ut på sidan. Det går då att skapa komplicerade layouter med kolumner och rader genom att gruppera innehållet i olika behållare som man sedan positionerar ut. Innan HTML5 fanns bara <div>-taggen att använda sig av för att uppnå detta, men med HTML5 kom många nya taggar som kan användas för att gruppera olika typer av sidinnehåll. Några exempel är <article>, <header> och <footer>.

Kortfattade beskrivningar av egenskaper, med exempel

Position

position

Denna egenskap används för att definiera typen av positioneringsmetod för ett element. Den kan innehålla något av värdena <static>, <absolute>, <relative>, <fixed>, <initial> eller <inherit>.

top

För absolut positionerade element sätter denna egenskap toppkanten för ett element till en enhet ovanför eller under toppkanten för dess närmaste förfader. Om elementet inte har någon förfader används istället body-elementet, och elementet följer då med vid scrollning av sidan. För relativt positionerade element sätter egenskapen toppkanten för ett element till en enhet ovanför eller under dess normala position. För statiskt positionerade element har egenskapen ingen effekt.

left

För absolut positionerade element sätter denna egenskap vänsterkanten för ett element till en enhet till vänster eller höger om vänsterkanten för dess närmaste förfader. Om elementet inte har någon förfader används istället body-elementet, och elementet följer då med vid scrollning av sidan. För relativt positionerade element sätter egenskapen vänsterkanten för ett element till en enhet till vänster eller höger om dess normala position. För statiskt positionerade element har egenskapen ingen effekt.

bottom

För absolut positionerade element sätter denna egenskap bottenkanten för ett element till en enhet ovanför eller under bottenkanten för dess närmaste förfader. Om elementet inte har någon förfader används istället body-elementet, och elementet följer då med vid scrollning av sidan. För relativt positionerade element sätter egenskapen bottenkanten för ett element till en enhet ovanför eller under dess normala position. För statiskt positionerade element har egenskapen ingen effekt.

right

För absolut positionerade element sätter denna egenskap högerkanten för ett element till en enhet till vänster eller höger om högerkanten för dess närmaste förfader. Om elementet inte har någon förfader används istället body-elementet, och elementet följer då med vid scrollning av sidan. För relativt positionerade element sätter egenskapen högerkanten för ett element till en enhet till vänster eller höger om dess normala position. För statiskt positionerade element har egenskapen ingen effekt.

z-index

Denna egenskap används för att definiera stapelordning för ett element. Ett element som har högre z-index än ett annat hamnar alltid framför det andra elementet. Egenskapen kan bara användas på element som är positionerade, med något av värdena absolute, relative eller fixed.

float

Denna egenskap används för att specificera om ett box-element ska flyta. Element placerade efter ett sådant element kommer att flöda runt det. För att undvika detta kan man använda egenskapen clear (se nedan).

clear

Denna egenskap används för att specificera på vilka sidor av ett element som flytande element inte ska kunna flöda.

Storlek

width

Denna egenskap används för att definiera bredden på ett element.

height

Denna egenskap används för att definiera höjden på att element.

max-width

Denna egenskap används för att definiera den maximala bredden för ett element. Det förhindrar att värdet på width-egenskapen blir större än den maxbredd som satts.

max-height

Denna egenskap används för att definiera den maximala höjden för ett element. Det förhindrar att värdet på height-egenskapen blir större än den maxhöjd som satts.

min-width

Denna egenskap används för att definiera den minimala bredden för ett element. Det förhindrar att värdet på width-egenskapen blir mindre än den minbredd som satts.

min-height

Denna egenskap används för att definiera den minimala höjden för ett element. Det förhindrar att värdet på height-egenskapen blir mindre än den minhöjd som satts.

vertical-align

Denna egenskap används för att definiera den vertikala justeringen av ett element.
